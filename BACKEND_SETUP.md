# Backend Setup — The Ardsley Carwash CMS
**Stack: PocketBase (CMS + API) + Render (hosting) + Netlify (frontend)**

---

## What You Get
| Feature | How |
|---|---|
| Admin dashboard | PocketBase UI at `yourdomain.onrender.com/_/` |
| Events management | Add/edit/delete events → auto-appears on site |
| Teams management | Add/edit/delete sponsored teams → auto-appears on site |
| Partners management | Add/edit/delete community partners → auto-appears on site |
| Booking records | View all form submissions with status tracking |
| Membership signups | Full signup records with vehicle info |
| Newsletter list | Collected email addresses |
| REST API | Auto-generated by PocketBase, no code needed |

---

## Step 1 — Download PocketBase

1. Go to https://pocketbase.io/docs/
2. Download the binary for your OS (Windows, Mac, or Linux)
3. Create a folder: `ardsley-backend/`
4. Place the `pocketbase` binary inside it

---

## Step 2 — Run Locally First

```bash
cd ardsley-backend
./pocketbase serve  
```

Open http://127.0.0.1:8090/_/ in your browser.
Create your admin account (email + password).

## Step 4 — Add Your First Events

In the PocketBase admin, go to **events** collection and add the events that are currently hardcoded in the HTML. Once you save them, the website will load them from the API automatically.

---

## Step 5 — Deploy PocketBase on Render

> **Cost:** Free web service (spins down after 15 min idle) + $0.25/mo disk for data persistence.
> Upgrade to a $7/mo instance if you need it always on.

---

### 5a — Create a Render account
Go to **https://render.com** and sign up (GitHub login recommended).

---

### 5b — Create a new Web Service

1. Dashboard → **New** → **Web Service**
2. Choose **"Deploy an existing image from a registry"**
3. Set the image URL to:
   ```
   ghcr.io/muchobien/pocketbase:latest
   ```
4. Click **Next** and fill in:

| Setting | Value |
|---|---|
| Name | `ardsley-carwash` |
| Region | Oregon (US West) or Ohio (US East — closest to NY) |
| Instance Type | Free |
| Port | `8090` |

5. Under **Environment Variables**, add:
   ```
   PORT = 8090
   ```

6. Click **Create Web Service**

---

### 5c — Add a Persistent Disk (required — without this, data is lost on restart)

1. Open your new service → **Disks** tab → **Add Disk**
2. Fill in:

| Setting | Value |
|---|---|
| Name | `pb-data` |
| Mount Path | `/pb/pb_data` |
| Size | 1 GB |

3. Click **Save** — Render will redeploy automatically

> **Note:** The 1 GB disk costs ~$0.25/month. This is required to keep your database alive across deploys and restarts.

---

### 5d — Set up your admin account

Once deployed, open your Render service URL:
`https://ardsley-carwash.onrender.com/_/`

Create your admin email + password on first visit.

> **If the admin UI doesn't let you create an account** (e.g. it's already been initialised or you forgot your password), use the Render Shell tab to run:
>
> ```bash
> /usr/local/bin/pocketbase superuser upsert your@email.com YourPassword123 --dir /pb_data
> ```
>
> The PocketBase binary in the `ghcr.io/muchobien/pocketbase:latest` image is located at **`/usr/local/bin/pocketbase`** and uses **`/pb_data`** as its data directory.
> (Not `/pb/pocketbase` or `/pb/pb_data` — those paths do not exist in this image.)

---

## Step 6 — Connect the Frontend

Open `js/config.js` and set your Render URL:

```js
POCKETBASE_URL: 'https://ardsley-carwash.onrender.com',
```

That's it — all forms and the events page will now talk to your live backend.

---

## Step 7 — Deploy Frontend to Netlify (Free)

1. Go to https://netlify.com and sign up
2. Drag and drop your `ardlseycarwash/` folder onto the Netlify dashboard
3. Your site goes live instantly at a `.netlify.app` URL
4. Connect a custom domain in Settings → Domain management

## CORS — Important

If the browser blocks requests to PocketBase, go to:
**PocketBase Admin → Settings → Application → Allowed CORS origins**

Add your frontend domain(s):
```
https://yoursite.netlify.app
https://ardsleycarwash.com
```

> Render also hosts the backend at `https://ardsley-carwash.onrender.com` — no need to add that to CORS since it's the API itself.
